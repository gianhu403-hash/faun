# Нормативная база протокола (Акт патрулирования лесов)

Документ обосновывает выбор формата, структуры и оформления PDF-протокола,
генерируемого AI-системой акустического мониторинга леса Faun.

## 1. Почему Акт патрулирования, а не Протокол об АП

AI-система юридически **не может** составлять Протокол об административном правонарушении
(ст. 28.2 КоАП РФ) — его составляет только уполномоченное должностное лицо
при непосредственном контакте с нарушителем.

Система Faun генерирует **Акт патрулирования лесов** — документ, прямо предусмотренный
Приказом Минприроды N 955 (Приложение 3), который:

- может быть составлен автоматически по результатам технического мониторинга
- является юридически значимым документом, прилагаемым к делу
- включает данные технических средств (БПЛА, аудиосенсоры)

| Нормативный акт | Положение |
|-----------------|-----------|
| КоАП РФ ст. 28.2 | Протокол об АП составляет **только** должностное лицо |
| Приказ Минприроды N 955 от 15.12.2021, п. 14 | Разрешает применение технических средств (БПЛА, аудиосенсоры) при патрулировании |
| Приказ Минприроды N 955, Приложение 3 | Утверждённая форма Акта патрулирования лесов |
| ПП РФ N 1098 от 30.06.2021, п. 38(2) | БПЛА включены в перечень технических средств лесного контроля |

## 5. Шрифт PT Serif

- Разработчик: Paratype (Россия)
- Лицензия: SIL Open Font License — свободная, разрешает bundling
- Метрически близок к Times New Roman
- Доступен в macOS, Linux (fonts-paratype), и бесплатно на fonts.paratype.com
- Альтернатива для госструктур: PT Astra Serif (метрический аналог TNR для Astra Linux)

## 6. Технический стек генерации PDF

```
Данные инцидента + Рейнджер + RAG-статьи
        ↓
Jinja2 (разделители: \VAR{}, \BLOCK{})
        ↓
act_patrol.tex (временный файл)
        ↓
lualatex -interaction=nonstopmode (2 прогона)
        ↓
PDF bytes → incident.protocol_pdf → Telegram-бот
```

Обоснование выбора LuaLaTeX:
- `babel` (не `polyglossia`) — рекомендация The LaTeX Companion 3rd ed. (2023)
- Нативная поддержка `fontspec`, `microtype`, переносов для русского языка
- Качество типографики: переносы, кернинг, лигатуры, микротипографика

## 8. AI/ML-данные как доказательства в судах РФ

- AI/ML-данные в российских судах — **ориентировочные** (вспомогательные) доказательства
- Основное доказательство — аудиозапись, фото с дрона, показания инспектора
- Цепочка: данные нейросети → проверка Рослесинфорг → выездная проверка → ГЛР
- Аналогия: данные видеокамер ГИБДД — принимаются при наличии метаданных и подписи инспектора
- Система «Лесохранитель»: 25 автоотчётов, 3000+ камер в 70 регионах, данные проверяются
- «Цифровая Земля» (Терра Тех): из 139 нейросетевых контуров — 3 подтверждённых нарушения

Условия принятия AI-данных судом:
1. Данные получены без нарушения законодательства (п. 14 Приказа N 955)
2. Запись содержит метаданные: дата, время, место
3. Система описана в материалах дела
4. Цепочка хранения не нарушена (hash-суммы, логи)

## 10. Документооборот: от AI-алерта до суда

```
Акустический сенсор → YAMNet v7 → TDOA → координаты
        ↓
    БПЛА → фото → Gemma 3 27B → анализ сцены
        ↓
    Faun генерирует:
    ├── Акт патрулирования лесов (Приказ N 955)     ← ЭТОТ ДОКУМЕНТ
    └── Telegram-алерт инспектору
        ↓
    Инспектор выезжает → осматривает → фото + описание
        ↓
    Faun дополняет акт данными инспектора → PDF
        ↓
    Если нарушитель установлен:
    ├── Инспектор составляет Протокол об АП (ст. 28.2 КоАП)
    ├── или передаёт материалы в полицию (ст. 260/258 УК)
    └── Данные Faun прилагаются как технические доказательства
```
