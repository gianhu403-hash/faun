# Промпт для браузерного агента: починка и переименование DataLens дашборда

## Контекст

Дашборд: https://datalens.yandex/aaamlpcpp7acu
Проект: Faun — AI-система акустического мониторинга леса.

Данные поступают через API Connector (подключение `faun-api-incidents`):
- URL: `http://81.85.73.178:8000/api/v1/datalens/incidents`
- Метод: GET, формат: JSON

---

## Промпт

```
Открой дашборд DataLens: https://datalens.yandex/aaamlpcpp7acu

На дашборде есть 2 сломанных виджета:
1. Чарт с ошибкой ERR.CHARTS.DATA_FETCHING_ERROR
2. Виджет «Типы нарушений» с ошибкой ERR.UNKNOWN / Internal error

Нужно выполнить 3 задачи: починить сломанные чарты, переименовать виджеты, убрать технические имена из счётчиков.

---

### Задача 1: Диагностика и починка сломанных чартов

Для каждого сломанного чарта:
1. Нажми «⋯» (три точки) → «Редактировать»
2. Проверь, к какому датасету привязан чарт
3. Проверь, все ли поля датасета доступны (нет ли красных/отсутствующих полей)
4. Если поле отсутствует — замени его на правильное из датасета

Доступные поля в датасете (API Connector):
- id (string) — UUID
- timestamp (datetime) — дата/время
- lat, lon (float) — координаты
- audio_class (string) — тип звука: chainsaw, gunshot, engine, axe, fire, background
- confidence (float) — уверенность 0.0–1.0
- gating_level (string) — alert, verify, log
- status (string) — pending, accepted, on_site, resolved, false_alarm
- district (string) — участковое лесничество
- response_time_min (float) — время реакции в минутах
- ranger_name (string) — имя инспектора
- resolution_details (string) — детали

Если чарт «Типы нарушений» не удаётся починить — удали его и создай новый:
- Тип: Столбчатая диаграмма
- X: audio_class
- Y: COUNT()
- Цвет: audio_class
- Название: «Структура нарушений»
- Сохрани и добавь на дашборд вместо сломанного

Если второй сломанный чарт — это «Статусы» (круговая), пересоздай:
- Тип: Круговая диаграмма
- Секции: status
- Значение: COUNT()
- Название: «Статусы инцидентов»

---

### Задача 2: Переименование виджетов

Зайди в редактирование дашборда (кнопка «Редактировать» вверху).
Переименуй каждый виджет (клик на название → изменить):

| Текущее название | Новое название |
|---|---|
| Всего инцидентов | Зарегистрировано инцидентов |
| Подтверждённых | Отработано инцидентов |
| Карта инцидентов | Геолокация инцидентов |
| Типы нарушений | Структура нарушений |
| Динамика за месяц | Динамика обнаружений |

Переименуй заголовки фильтров (селекторов):

| Текущее | Новое |
|---|---|
| audio_class | Класс звука |
| status | Статус |
| district | Район |
| timestamp | Период |

---

### Задача 3: Убрать технические имена из счётчиков

Счётчики «Зарегистрировано инцидентов» и «Отработано инцидентов» показывают технические имена полей (total_count, resolved_count) как подпись.

Для каждого счётчика:
1. Нажми «⋯» → «Редактировать»
2. В настройках чарта найди подпись/label и удали техническое имя
3. Оставь только число (значение)
4. Сохрани

---

### Проверка

После всех изменений:
1. Сохрани дашборд
2. Обнови страницу (F5)
3. Убедись, что:
   - Все чарты загружаются без ошибок
   - Названия виджетов обновлены
   - Счётчики показывают только числа без технических имён
   - Фильтры подписаны по-русски
4. Сделай скриншот результата
```
